# AI 코드 어시스턴스 교육 프로그램 판매 사이트 설계서

작성일: 2026-04-19  
범위: 1차 오픈(1주 이내) 정적 랜딩페이지

## 1. 목표
- 비즈니스 목표: AI 코드 어시스턴스 교육 프로그램 판매를 위한 상담 전환 확보
- 1차 핵심 KPI: 주간 상담 신청 건수
- 전환 흐름: 상담 신청(카카오톡) → 상담 진행 → 계좌이체 안내 및 결제

## 2. 타깃 사용자
- 주요 타깃: 40대 이상 성인
- 기대 사용자 특성:
  - 기술 용어보다 실용성과 신뢰를 중시
  - 빠른 화려함보다 안정감 있는 정보 구조를 선호
  - 실제 사례를 통해 학습 효과를 확인하려는 경향

## 3. 접근 방식(확정)
- 확정안: 단일 롱스크롤 정적 랜딩페이지(A안)
- 선택 이유:
  - 1주 이내 제작/검수 가능
  - 사용자 동선을 단순화하여 상담 전환 집중 가능
  - 초기 운영/수정 비용 최소화

## 4. 정보구조 및 사용자 흐름
- 페이지 구조: 단일 페이지(`index.html`) + 섹션 앵커 이동
- 기본 흐름:
  1) 프로그램 이해
  2) 신뢰 형성(후기/사례)
  3) 카카오톡 상담 신청
  4) 상담 후 계좌이체 결제
- CTA 원칙: 상단/중단/하단의 모든 CTA를 카카오 상담으로 통일

## 5. 화면 구성
1) 히어로 섹션
- 메시지: 40대 이상 성인을 위한 AI 코드 어시스턴스 교육
- 톤: 신뢰감 있는 교육기관형(차분/정돈)
- 요소: 핵심 카피, 간단한 가치 제안, 카카오 상담 버튼

2) 프로그램 소개/커리큘럼(최우선 섹션)
- 구성: 초급 → 활용 → 실전
- 각 단계별 학습 결과를 짧고 명확하게 제시
- 불필요한 기술 과시는 배제하고 실무/생활 적용 포인트 중심

3) 수강생 후기/사례(핵심 신뢰 섹션)
- 형식: Before → After
- 권장 정보: 연령대/배경, 문제 상황, 수강 후 변화
- 목적: 유사 연령 사용자에게 신뢰 전이

4) 진행 방식
- 단계: 상담 신청 → 학습 진단 → 수강 진행 → 상담 후 계좌이체 안내
- 결제는 상담 이후 진행됨을 명확히 표기

5) FAQ
- 우선 항목: 난이도, 준비물, 수강시간, 환불 정책
- 문체: 짧고 단정한 문장, 모호한 표현 금지

6) 하단 CTA
- 고정 또는 시각적으로 분명한 카카오 상담 신청 버튼
- 중복 클릭 유도를 위한 동일 액션 유지

## 6. 데이터 및 기능 동작
- 데이터 소스: 정적 콘텐츠(HTML 내 텍스트)
- 상호작용: CTA 클릭 시 카카오 상담 링크 이동
- 결제 처리: 사이트 내 결제 기능 미구현(상담 후 계좌이체 안내)
- 추적(권장): CTA 클릭 수 측정을 위한 경량 분석 스크립트 1종

## 7. 오류/품질 기준
- 링크 품질:
  - 카카오 링크 누락/오타 0건
  - 모든 CTA에서 동일 링크 동작
- 반응형 품질:
  - 모바일에서 CTA 버튼 가시성 및 터치 영역 확보
- 콘텐츠 품질:
  - 대상 연령에 맞는 쉬운 문장
  - 과장 표현 지양, 신뢰 중심 문구 유지

## 8. 범위(1차 오픈)
포함:
- 단일 정적 랜딩페이지 1개
- 핵심 섹션 6개(히어로, 커리큘럼, 후기, 진행 방식, FAQ, CTA)
- 카카오 상담 유도 동선

제외:
- 회원가입/로그인
- 사이트 내 결제 시스템
- 관리자 페이지
- 다국어/블로그/콘텐츠 CMS

## 9. 오픈 체크리스트
- [ ] 카카오 상담 링크 최종 반영
- [ ] CTA 동작 상/중/하단 점검
- [ ] 모바일(대표 해상도) 가독성 점검
- [ ] 후기 문구 사실성/표현 점검
- [ ] FAQ 핵심 4개 항목 검수

## 10. 성공 판단
- 1차 판단 지표: 주간 상담 신청 건수
- 운영 관점 보조 지표(선택): CTA 클릭 수, 섹션별 이탈 구간
